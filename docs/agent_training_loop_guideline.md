# 로봇팔 강화학습 성공 조건

로봇팔 정책은 최종 위치만 맞추는 것이 아니라, 물체를 안정적으로 집고 목표 위치에 자연스럽게 내려놓아야 성공으로 인정한다.

## 에이전트 운영 원칙

- 에이전트는 이 문서의 성공 조건과 실패 조건을 우선 기준으로 삼는다.
- pick-and-place 보상함수, 성공 gate, telemetry, visual approval 기준을 바꿀 때는 먼저 `docs/r30o_pick_place_reward_design_db.md`를 참고한다.
- 현재 전체 목표는 "학습 로드맵"으로 관리한다. 1단계는 단일 랜덤 물체 pick-and-place, 2단계는 두 물체 cued placement, 3단계는 두 물체 randomized placement, 4단계는 물체 수를 늘리는 incremental `n`-object placement, 5단계는 최종 `n`-object generalization이다.
- 현재 활성 단계는 학습 로드맵 1단계이며, 1단계가 numeric, telemetry, video gate를 모두 통과하면 다음 단계로 넘어간다.
- 사용자가 세부 실험값을 하나하나 지정하지 않아도, 에이전트가 현재 학습 상태를 확인하고 다음 평가, 수정, 검증 단계를 직접 결정한다.
- 실행 중인 학습은 live process, checkpoint, TensorBoard log, rollout video, telemetry를 확인한 뒤 판단한다.
- `success_rate`만으로 완료를 선언하지 않는다.
- 의미 있는 결정, 실행, 평가, 실패, 남은 문제는 `docs/recording_handoff_log.md`에 남긴다.

## 환경 배치

성공 조건은 아래 배치를 기준으로 판단한다.

- 로봇은 Fetch 로봇팔이다.
- 물체는 작은 정육면체 블록이다.
- 물체는 로봇 정면 테이블 위에서 시작한다.
- 목표 박스는 로봇이 정면을 바라봤을 때 기준 왼쪽 측면에 놓인 물리 bin이다.
- 목표 지점은 bin의 공중 중심이 아니라 bin 내부 바닥 근처다.
- bin은 바닥과 네 벽을 가진 열린 상자이며, 물체는 벽에 기대거나 튕겨 들어가는 것이 아니라 내부 바닥에 안정적으로 놓여야 한다.
- 물체는 로봇 정면 작업공간 안의 일정 범위에서 랜덤 생성되어야 하며, 고정된 움직임 하나로 풀 수 없어야 한다.
- 로봇은 정면의 랜덤 물체를 집어 왼쪽 측면 bin 안으로 옮겨야 한다.
- 성공은 로봇 기준 `정면 테이블 랜덤 물체 -> 왼쪽 측면 bin`으로 물체가 이동했을 때만 인정한다.

## 성공 행동

성공한 episode는 다음 흐름을 보여야 한다.

1. 로봇팔이 물체에 접근한다.
2. gripper가 물체를 안정적으로 집는다.
3. 물체를 목표 영역까지 운반한다.
4. 목표 영역 안 또는 바로 위에서 물체를 바닥 근처까지 낮춘다.
5. 낮은 위치에서 gripper를 연다.
6. 물체가 목표 영역 안에 낮고 안정적으로 정착한다.
7. gripper가 물체에서 물러난다.

## 성공 판정 조건

성공으로 인정하려면 아래 조건을 모두 만족해야 한다.

- 물체가 목표 영역 안에 있어야 한다.
- 물체가 목표 영역의 바닥 근처에 있어야 한다.
- 물체 중심이 bin 벽 안쪽 여유 공간에 들어와 있어야 한다.
- 물체 속도가 충분히 낮아 안정적으로 멈춘 상태여야 한다.
- gripper가 물체를 낮은 위치까지 운반한 뒤 열려야 한다.
- gripper가 열린 뒤 물체에서 충분히 멀어져야 한다.
- 물체가 벽이나 구조물에 강하게 부딪히지 않아야 한다.
- 목표 영역에 들어간 뒤 빠르게 튀거나 흔들리지 않아야 한다.
- episode 중 투척성 움직임이 없어야 한다.

## 실패로 볼 행동

아래 행동은 최종 위치가 맞더라도 실패로 본다.

- 물체를 던져서 목표 영역에 넣는다.
- 물체가 벽이나 구조물에 맞고 들어간다.
- 높은 위치에서 떨어뜨린 뒤 우연히 목표 영역 안에 멈춘다.
- 물체가 bin 벽 위나 모서리에 걸친다.
- 물체 중심은 목표 근처지만 실제로 bin 내부 바닥에 놓이지 않는다.
- 목표 영역 안에서 빠르게 튀거나 미끄러진다.
- gripper가 물체를 계속 잡고 있어 실제로 놓지 않는다.
- gripper가 닫힌 채 낮은 위치에서 버틴다.
- 목표 영역 위에서 물체를 높게 들고 멈춘다.
- 물체가 목표 영역 뒤쪽으로 과하게 넘어간다.
- 성공률은 높지만 영상에서 조작이 자연스럽지 않다.

## 완료 기준

학습 완료는 숫자 지표만으로 판단하지 않는다. 성공률이 높더라도 영상과 상태 정보가 같은 결론을 보여야 한다.

완료로 인정하려면 다음이 모두 확인되어야 한다.

- 영상에서 `접근 -> 집기 -> 운반 -> 낮추기 -> 열기 -> 정착 -> 물러남` 순서가 보인다.
- 물체가 목표 영역 안에서 낮고 안정적으로 멈춘다.
- gripper가 실제로 열리고 물체에서 물러난다.
- 던지기, 튐, 벽 접촉, high hover, closed hold가 없다.
- 상태 정보가 안정 정착과 정상 release를 지지한다.
